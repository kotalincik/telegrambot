## 4. Предметная область и аналоги

### 4.1 Предметная область

Стоматологические клиники ежедневно обрабатывают десятки записей пациентов. Традиционная схема записи по телефону имеет ряд недостатков:
- Ограниченное время работы регистратуры
- Длинные очереди на линии
- Человеческий фактор (ошибки при записи)
- Необходимость держать штат операторов

**Автоматизация через Telegram-бота позволяет:**
- Записываться круглосуточно
- Видеть актуальное расписание врачей
- Получать напоминания о приёме
- Управлять записями без звонков

### 4.2 Акторы системы

| Актор | Роль | Функции |
|-------|------|---------|
| **Пациент** | Клиент клиники | Запись на приём, просмотр записей, отмена |
| **Врач** | Медицинский работник | Работа с расписанием, приём пациентов |
| **Администратор** | Управляющий | Управление всей системой, статистика |

### 4.3 Аналоги

| Продукт | Особенности | Недостатки |

## 5. Выбор стека технологий

### 5.1 Обоснование выбора

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| **Язык** | Python 3.11 | Простота, богатая экосистема, async/await |
| **Bot Framework** | aiogram 3.x | Асинхронность, FSM, современный синтаксис |
| **Web Framework** | FastAPI | Высокая производительность, автодокументация |
| **ORM** | SQLAlchemy 2.0 | Декларативные модели, async поддержка |
| **База данных** | PostgreSQL + asyncpg | Надёжность, типы JSON, async драйвер |
| **Миграции** | Alembic | Интеграция с SQLAlchemy, версионирование |
| **Тестирование** | pytest + pytest-asyncio | Стандарт для Python, async тесты |
| **Конфигурация** | pydantic-settings | Валидация, .env поддержка |

### 5.2 Альтернативы и почему отклонены

- **Django** — слишком тяжёлый для этого проекта, нет встроенной async поддержки ORM
- **python-telegram-bot** — менее элегантный синтаксис FSM
- **Flask** — нет встроенной валидации и async
- **SQLite** — не подходит для конкурентного доступа в async режиме

## 7. Описание реализации

### 7.1 Модели данных (SQLAlchemy)

Реализовано 8 моделей с полной типизацией:
- Наследование от `DeclararativeBase`
- Использование `Mapped[]` и `mapped_column()` (SQLAlchemy 2.0 style)
- Enum-классы для статусов и специализаций
- Relationship между сущностями
- Индексы для оптимизации запросов

### 7.2 Репозитории (CRUD)

5 репозиториев для основных операций:
- `PatientRepository` — управление пациентами
- `DoctorRepository` — управление врачами
- `ServiceRepository` — управление услугами
- `AppointmentRepository` — управление записями
- `ScheduleRepository` — управление расписанием

### 7.3 Сервисный слой

2 основных сервиса:
- `BookingService` — логика записи на приём
- `ScheduleService` — работа с расписанием, генерация слотов

### 7.4 Telegram-бот

**FSM (Finite State Machine) для записи:**
1. `selecting_specialization` → выбор специализации
2. `selecting_doctor` → выбор врача
3. `selecting_service` → выбор услуги
4. `selecting_date` → выбор даты
5. `selecting_time` → выбор времени
6. `confirming` → подтверждение

**Основные команды:**
- `/start` — начало работы, регистрация
- `/help` — справка
- `/menu` — главное меню

**Функционал:**
- Запись на приём (пошаговый wizard)
- Просмотр записей с возможностью отмены
- Профиль пациента
- Админ-панель для управления

### 7.5 REST API (FastAPI)

**Endpoints:**
- `GET /api/doctors` — список врачей
- `GET /api/doctors/{id}/schedule` — расписание
- `GET /api/appointments` — список записей
- `POST /api/appointments/{id}/cancel` — отмена
- `GET /api/statistics` — статистика клиники

### 7.6 Алгоритм генерации доступных слотов

```python
async def get_available_slots(doctor_id, date):
    # 1. Проверяем исключения (отпуск, больничный)
    exceptions = await get_exceptions(doctor_id, date)
    
    # 2. Получаем базовое расписание на день недели
    schedule = await get_schedule(doctor_id, day_of_week)
    
    # 3. Получаем существующие записи
    appointments = await get_appointments(doctor_id, date)
    
    # 4. Получаем перерывы
    breaks = await get_breaks(doctor_id, day_of_week)
    
    # 5. Генерируем слоты с проверкой пересечений
    for slot in generate_slots(schedule):
        if not intersects(slot, breaks):
            if not intersects(slot, appointments):
                yield slot
```
